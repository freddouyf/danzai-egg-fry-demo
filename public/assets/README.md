# 旦仔 PNG 素材说明

把设计师提供的 PNG 图片放在这个目录中，刷新浏览器后游戏会自动优先使用它们。没有对应图片时，游戏会继续使用自带的 Canvas/CSS 占位形象，不会报错或白屏。

## 当前已经接入的设计师素材

游戏会直接使用 `origin/` 中的基础表情：

| 当前文件 | 游戏用途 |
| --- | --- |
| `origin/资源 4.png` | 开始页爱心旦仔 |
| `origin/资源 6.png` | 游戏中的默认旦仔 |
| `origin/资源 7.png` | 翻面/奋斗状态 |
| `origin/资源 3.png` | 高分和开心状态 |
| `origin/资源 1.png` | 糊锅和低分反馈 |
| `origin/资源 2.png` | 鼓励状态预留 |
| `origin/资源 5.png` | 冥想状态预留 |

`special/` 中的 7 套服装不会在当前版本随机出现，它们已经登记在 `src/skins.js`，后续用于金币购买和衣柜换装。

AI 生成的手绘厨房背景位于 `generated/kitchen-stage.png`。游戏中的锅、鸡蛋、火焰和粒子仍由 Canvas 实时绘制，以便随火候和事件动态变化。

## 推荐文件名

| 文件名 | 用途 | 是否必需 |
| --- | --- | --- |
| `danzai_idle.png` | 开始页和游戏中的默认旦仔 | 推荐至少提供这一张 |
| `danzai_happy.png` | 完美出锅或高分结算 | 可选 |
| `danzai_flip.png` | 翻面瞬间的动作 | 可选 |
| `danzai_fail.png` | 糊锅或低分结算 | 可选 |
| `pan.png` | 游戏中的平底锅 | 可选 |
| `egg.png` | 锅里的鸡蛋 | 可选 |
| `logo.png` | 开始页标题 Logo | 可选 |

角色图片建议使用透明背景 PNG，尺寸至少为 512×512 像素，并在图片四周留少量空白，避免手脚被裁掉。

## 文件名不一样怎么办

不必强行改名。打开 `src/assets.js`，在对应项目的数组中加入实际文件名即可。例如设计师给你的文件叫 `旦仔开心最终版.png`：

```js
danzaiHappy: ["danzai_happy.png", "旦仔开心最终版.png"],
```

加载器会从左到右尝试文件名，并使用第一张成功找到的图片。修改后保存，Vite 会自动刷新页面。

## 使用提示

- PNG 可以保留中文文件名，推荐英文小写文件名只是为了后续协作更方便。
- 替换图片后若没有立即更新，可在浏览器中按 `Ctrl + F5` 强制刷新。
- Logo 可以是横版或方形，页面会按比例缩放，不会拉伸。
- 平底锅和鸡蛋素材最好使用透明背景，并让主体位于画面中央。
